# UAT Results - 2026-03-28

## Status: PARTIAL - Only leaves rendered

## Observations

- Metal terrain renderer activates successfully (profiler shows draws, verts, GPU time)
- IOSurface compositing pipeline works (confirmed via automated pixel reads and screenshots)
- **Only leaf blocks rendered correctly** - other block types (dirt, grass, stone, wood) did not appear
- The automated screenshots during development showed green leaf textures on the left side of screen, which matched this UAT finding
- Streaks/artifacts still visible for non-leaf geometry

## Likely Root Cause

The issue is likely related to which render types actually have valid VBO data at `RenderWorldLastEvent` time:
- `cutout` render type (leaves) may retain VBO data longer than `solid` (dirt, stone, grass)
- The mixin cancels `renderChunkLayer` which may affect VBO lifecycle differently per render type
- GL buffer size was 0 for some VBOs (verified in diagnostics) - only certain chunks/render types had valid data

## Next Steps (DO NOT START - game is running)

1. Investigate per-render-type VBO validity - which types have data at event time
2. Consider capturing VBO data DURING `renderChunkLayer` via mixin (before cancel) instead of after
3. Alternative: let GL render terrain normally (don't cancel via mixin), then overlay Metal terrain on top
4. The mixin cancellation strategy may need to be rethought entirely - perhaps Metal should intercept the upload step, not the draw step
